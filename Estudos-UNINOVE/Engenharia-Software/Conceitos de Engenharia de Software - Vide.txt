Aspectos da Engenharia de Software

Ciclo de vida
Especificações - Análise - Implementação - Validação - Entrega

Modelos de processos e desenvolvimento
Cascata - Modelo que segue o padrão de especificar, analisar, implementar, validar e finaliza na entrega, sem haver retorno.
Evolucionário - Tenta resolver/solucionar uma especificação muito complexa com os modos:
		Exploratório ( sugere que faça o desenolvimento em cima de uma especificação não fechada, vai sendo explorada até que tenha o projeto finalizado - código bagunçado ) ou por Prototipação - Usa o conceito evolucionario para levantar informações que serão uteis na continuidade do seu processo )
Espiral - Não só desenvolver o projeto mas com gestão do projeto ( analise dos riscos por ex, sugere que especifique, analise, implemente, valide mas sempre a um planejamento a uma analise de risco. )
Incremental - quebrar o dev em partes - realiza um ciclo e entrega, tipo 1.0 2.0 eetc.
